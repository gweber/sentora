# Security Audit — March 2026

## Scope
Full codebase audit of all backend domains, middleware, utilities, frontend stores,
composables, views, and infrastructure configuration. The sync pipeline was excluded
(audited separately).

## Methodology
Systematic file-by-file review covering: error handling, race conditions, dead code,
duplicated logic, data integrity, input validation, authentication/authorization,
frontend state management, performance, and configuration.

## Summary
- **50 findings** identified and remediated
- **5 Critical**, **8 High**, **25 Medium**, **12 Low** severity
- All findings fixed in-place with no breaking API changes

## Critical Findings (Fixed)
1. Tenant isolation bypass via `X-Tenant-ID` header manipulation
2. Weak key derivation (unsalted SHA-256) for field encryption
3. JWT signing key reused as encryption key
4. SAML account hijack via hardcoded `email_verified`
5. Audit chain genesis race condition

## Key Improvements
- HKDF-SHA256 key derivation with domain separation
- PKCE support for OIDC flows
- DNS rebinding protection in SSRF validator
- Violation count caps in compliance engine
- defusedxml for XML parsing
- Non-root nginx container
- MongoDB authentication in Docker Compose

## Recommendations for Future Work
- Migrate to MongoDB-backed rate limiting for login endpoints
- Add E2E tests to CI pipeline
- Consider HttpOnly cookie storage for JWT tokens
- Plan migration to remove legacy plaintext field fallback in crypto.py
